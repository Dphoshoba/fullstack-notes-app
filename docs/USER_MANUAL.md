# User Manual

This manual explains how to use the multilingual Notes workspace app. It is written for beginners and covers the main workflows you will use every day.

## Quick Start

1. Register for an account or log in.
2. Open the Dashboard.
3. Create your first note.
4. Add a category and tags so the note is easy to find later.
5. Pin or star important notes.
6. Use search, filters, and smart chips to narrow the list.
7. Try AI Tools when you want a summary, tag suggestions, or meeting help.
8. Use Settings to update your profile, language, and workspace options.

## Register

Use the Register page to create a new account.

1. Enter your name.
2. Enter your email address.
3. Enter a password.
4. Select Create account.

After registration, you can log in and open your protected Dashboard.

## Login and Logout

Use the Login page when you already have an account.

1. Enter your email address.
2. Enter your password.
3. Select Login.

Use Log out in the Dashboard header when you are finished. Logging out removes your current session from the browser.

## Dashboard

The Dashboard is the main place where you work with notes.

The header includes:

- Language selector
- Refresh
- Export
- Profile
- Guide
- Settings
- Admin, for admin and superadmin users only
- Log out

The top of the Dashboard also shows stat cards for your loaded notes, pinned notes, starred notes, categories, and user role.

## Standard Notes

Standard notes are regular personal or workspace notes.

To create a standard note:

1. Leave Note type set to Standard.
2. Add a title.
3. Add the note body.
4. Choose a category.
5. Add tags if needed.
6. Choose Private or Workspace if you belong to a workspace.
7. Choose whether the note should be pinned or starred.
8. Select Create note.

Private notes are visible only to you. Workspace notes are visible to members of your workspace.

## Meeting Notes

Meeting notes are an extension of the normal notes system. They keep the same title, body, category, tags, visibility, pin, and star options, but also support meeting details.

To create a meeting note:

1. Set Note type to Meeting.
2. Add the meeting date if known.
3. Add attendees separated by commas.
4. Add a follow-up date if needed.
5. Select Use meeting template to prefill the body with Agenda, Discussion, Decisions, Action items, and Next steps.
6. Finish the note like a standard note.

Meeting note cards show a Meeting badge. Meeting notes can also show a Meeting summary with date, attendees, agenda, decisions, follow-up date, and action items when those details exist.

## Create, Edit, and Delete Notes

Create notes from the Create note panel on the Dashboard.

Use Edit on a note card to update:

- Title
- Body
- Category
- Tags
- Note type
- Meeting details
- Visibility
- Pinned status
- Starred status

Use Delete to remove a note. The app asks for confirmation first so you do not delete a note by mistake.

## Categories and Tags

Categories are broad groups, such as General, Work, Personal, Ideas, or Tasks.

Tags are flexible labels. Add multiple tags separated by commas. Search can match tags, so good tags make notes easier to find.

## Pin and Star

Pin important notes to keep them near the top of the notes list. Pinned notes appear before unpinned notes, and each group is sorted newest first.

Star notes you want to find quickly. Use the Favorites filter or Starred smart chip to show only starred notes.

## Search, Filters, and Smart Chips

Use the search bar above the notes list to search by title, body, and tags. Search updates the results as you type.

Use filters to narrow notes by:

- Scope: all, private, or workspace notes
- Note type: all, standard, or meeting notes
- Category
- Favorites

Use smart chips for quick filtering:

- Pinned
- Starred
- Workspace
- Private
- This week

Search and filters work with pagination. Changing a filter resets the list to page 1.

## Comments

Open Comments / Attachments on a note card to view or add comments.

You can:

- Post a new comment
- Edit your own comments
- Delete your own comments

Workspace owners and managers can delete comments on workspace notes when moderation is needed.

## Attachments

Open Comments / Attachments on a note card to upload, download, or delete files.

Supported file types:

- PDF
- JPG, JPEG, PNG, WEBP
- DOC, DOCX
- TXT
- MD

Each file can be up to 10MB.

Private note attachments are visible only to the note owner. Workspace note attachments are visible to members of the same workspace. The uploader, note owner, workspace owner, or workspace manager can delete attachments when allowed.

## Workspaces

A workspace is a shared area for an office or team. It lets people create shared workspace notes while still keeping private notes separate.

Workspace roles:

- Owner: creates and controls the workspace, manages members, and can manage shared workspace notes.
- Manager: can view members, invite people, add members, and help manage shared workspace notes.
- Staff: can create workspace notes and manage their own workspace notes.

## Office and Team Use

Staff can use workspaces to keep shared notes in one place.

Example uses:

- Meeting notes for team discussions
- Office procedures
- Client or project notes
- Shared task notes
- Files attached to team notes
- Comment threads for follow-up questions

Use Private notes for personal drafts or sensitive notes. Use Workspace notes when the note should be visible to the team.

## Invitations

Workspace owners and managers can create invitation links from Settings.

1. Open Settings.
2. Go to the Workspace section.
3. Enter the invitee email address.
4. Choose Staff or Manager.
5. Select Create invite.
6. Copy the invite link and send it to the invited person.

The invited person must log in with the same email address that was invited. Invites expire after seven days or after they are accepted.

## Settings

Use Settings to manage account and workspace preferences.

You can update:

- Name
- Preferred language
- Default note scope

You can also view:

- Email
- App role
- Plan
- Workspace role
- Workspace members, if your workspace role allows it

## Profile

Use Profile in the Dashboard header to view your name, email, and role.

Select Edit Profile to update your name. Email and role cannot be changed from the profile panel.

## Language Selector

Use the language selector in the Dashboard header to change the app language. The selected language is stored in your browser so it is remembered later.

The app supports English, French, Spanish, Portuguese, Arabic, and Krio/Sierra Leone Creole.

## AI Tools and Usage Limits

AI Tools help you work faster with notes.

Available AI actions include:

- Summarize a note
- Suggest tags
- Generate smart insights
- Convert to meeting minutes
- Extract action items
- Extract attendees and decisions

Select a note before running note-specific AI actions.

Free plans include limited AI usage. The Plan & Usage card shows how many AI uses remain. Premium gives more AI usage for summaries, tag suggestions, meeting actions, and insights.

## Saving AI Results

AI results appear in the AI result panel before you save them.

You can:

- Save to note
- Save as comment
- Replace note body, for meeting AI results
- Append to note body
- Save to meeting details
- Cancel

The app does not silently overwrite note content. You choose how the AI result should be saved.

## Meeting AI Actions

Meeting AI actions help turn rough notes into structured meeting notes.

Use:

- Convert to meeting minutes to clean up raw meeting notes.
- Extract action items to identify tasks, owners, due dates, and status.
- Extract attendees and decisions to fill meeting details.

Save to meeting details changes the note type to Meeting and stores the structured meeting data in the note.

## Export

Use Export in the Dashboard header to download currently visible notes.

Export formats:

- Markdown
- JSON

Standard notes export with normal note details. Meeting notes export with structured meeting sections, including agenda, discussion, decisions, action items, attendees, and follow-up date.

## Plan, Upgrade, and Billing

The Plan & Usage card shows:

- Current plan
- AI usage
- Remaining AI uses
- Upgrade button for free users
- Manage billing button for premium users

Use Upgrade to open Stripe Checkout. Use Manage billing to open the Stripe billing portal when you have a premium plan.

Plan changes should come from Stripe billing events, not from manual frontend changes.

## Admin and Superadmin

App roles are separate from workspace roles.

App roles:

- user: can use notes, profile, settings, workspaces, comments, attachments, AI tools, billing, export, and language features.
- admin: can open the Admin panel and view users.
- superadmin: can view users and change user roles.

Admins and superadmins see an Admin button in the Dashboard header. Superadmins can change another user's role, but cannot change their own role from the modal.

## Before Launch Checklist

Use this checklist before deploying to production.

- Add all backend environment variables in Render.
- Add `VITE_API_BASE_URL` in Netlify.
- Use strong production JWT secrets.
- Set `COOKIE_SECURE=true` in production.
- Set `CLIENT_ORIGIN` to the exact Netlify frontend URL.
- Set MongoDB Atlas Network Access for your deployment.
- Use a MongoDB database user with a strong password.
- Use Stripe live keys only when you are ready for real payments.
- Set the Stripe webhook endpoint to `/api/billing/webhook`.
- Add the live Stripe webhook secret to Render.
- Test register, login, logout, create note, edit note, delete note, comments, attachments, AI tools, upgrade, billing portal, admin, and workspace flows.
- Never commit real secrets to Git.

## Troubleshooting

### I cannot log in

Check your email and password. Confirm the backend server is running and the frontend is using the correct API URL.

### Notes are not loading

Select Refresh on the Dashboard. If notes still do not load, confirm the backend and MongoDB connection are running.

### Search is not finding a note

Try a shorter keyword. Clear category, favorites, scope, note type, and smart chip filters.

### I cannot see workspace notes

Open Settings and confirm you belong to a workspace. Private notes and workspace notes have separate visibility rules.

### I cannot upload an attachment

Check the file type and size. The file must be one of the supported types and must be 10MB or smaller.

### Comments or attachments do not load

Open the Comments / Attachments section again. If the problem continues, refresh the Dashboard and check that you still have access to the note.

### AI Tools are disabled

You may have reached the Free plan AI usage limit. Check Plan & Usage or upgrade to Premium.

### Upgrade does not open Stripe

Confirm the backend has Stripe environment variables configured. In production, also confirm the deployed frontend and backend URLs are correct.

### Manage billing does not open

Manage billing is available for premium users with an active Stripe customer. Confirm Stripe Billing Portal is configured.

### I cannot see the Admin button

Only admin and superadmin users can see Admin tools.

### My language did not change

Use the language selector again. Refresh the page if your browser cached the old view.
